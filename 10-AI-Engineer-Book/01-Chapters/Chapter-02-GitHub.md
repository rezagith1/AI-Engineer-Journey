# GitHub

---

## GitHub چیست؟

GitHub یک سرویس مبتنی بر Git است که برای نگهداری، مدیریت و اشتراک‌گذاری پروژه‌های نرم‌افزاری روی اینترنت استفاده می‌شود.

به بیان ساده، اگر Git را موتور مدیریت نسخه بدانیم، GitHub خانه‌ای است که پروژه‌های Git در آن نگهداری می‌شوند.

GitHub علاوه بر ذخیره پروژه‌ها، امکان همکاری تیمی، بررسی تغییرات، مدیریت نسخه‌ها و انتشار پروژه‌ها را نیز فراهم می‌کند.

---

## تفاوت Git و GitHub

بسیاری از افراد در ابتدای یادگیری این دو مفهوم را با یکدیگر اشتباه می‌گیرند، در حالی که وظایف آن‌ها کاملاً متفاوت است.

| Git | GitHub |
|------|---------|
| یک نرم‌افزار مدیریت نسخه است. | یک سرویس میزبانی پروژه‌های Git است. |
| روی کامپیوتر نصب می‌شود. | روی اینترنت قرار دارد. |
| تاریخچه تغییرات پروژه را مدیریت می‌کند. | پروژه‌ها را نگهداری، نمایش و اشتراک‌گذاری می‌کند. |
| بدون اینترنت نیز قابل استفاده است. | برای استفاده به اینترنت نیاز دارد. |

---

## Local Repository

Repository محلی (Local Repository)، نسخه‌ای از پروژه است که روی کامپیوتر شخصی قرار دارد.

تمام تغییراتی که ابتدا ایجاد می‌کنیم، در Repository محلی ذخیره می‌شوند.

در این پروژه، مسیر Repository محلی من به صورت زیر است:

```text
E:\AI-Engineer-Journey
```

---

## Remote Repository

Remote Repository نسخه‌ای از پروژه است که روی یک سرور مانند GitHub قرار دارد.

این نسخه معمولاً برای نگهداری، پشتیبان‌گیری و همکاری با سایر برنامه‌نویسان استفاده می‌شود.

در این پروژه، Remote Repository من عبارت است از:

```text
https://github.com/rezagith1/AI-Engineer-Journey
```

---

## Remote چیست؟

Git می‌تواند به یک یا چند Repository روی اینترنت متصل شود.

به هر یک از این اتصال‌ها یک **Remote** گفته می‌شود.

هر Remote دارای یک نام و یک آدرس است.

---

## Origin چیست؟

Origin نام مستعار (Alias) پیش‌فرض برای اولین Remote در Git است.

به جای نوشتن کامل آدرس GitHub در هر دستور، از نام کوتاه **origin** استفاده می‌کنیم.

به عنوان مثال:

```bash
git remote add origin https://github.com/rezagith1/AI-Engineer-Journey.git
```

در این دستور، Git یاد می‌گیرد که هر زمان نام **origin** را ببیند، منظور همان آدرس Repository در GitHub است.

---

## Branch چیست؟

Branch به معنای شاخه‌ای از پروژه است.

هر پروژه Git حداقل یک شاخه اصلی دارد.

در این پروژه، شاخه اصلی با نام **master** ایجاد شده است.

در بسیاری از پروژه‌های جدید، نام شاخه اصلی **main** است.

---

## Push چیست؟

Push عملیاتی است که تغییرات Repository محلی را به Repository آنلاین ارسال می‌کند.

نمونه دستور:

```bash
git push -u origin master
```

این دستور باعث می‌شود آخرین تغییرات پروژه روی GitHub منتشر شوند.

---

## Fetch چیست؟

Fetch عملیاتی است که اطلاعات و تغییرات موجود در Repository آنلاین را بدون اعمال روی پروژه محلی دریافت می‌کند.

به عبارت دیگر، Fetch فقط اطلاعات را دریافت می‌کند و هنوز فایل‌های پروژه را تغییر نمی‌دهد.

---

## Authentication

قبل از اینکه Git بتواند پروژه‌ای را روی GitHub ارسال کند، باید هویت کاربر تأیید شود.

به این فرآیند Authentication گفته می‌شود.

در ویندوز معمولاً این کار توسط Git Credential Manager انجام می‌شود.

## نکات کلیدی

- Git تاریخچه پروژه را مدیریت می‌کند.

- Origin نام مستعار Repository آنلاین است.

- هر پروژه می‌تواند دارای چند Branch باشد.
- Authentication امنیت ارتباط بین Git و GitHub را تضمین می‌کند.
