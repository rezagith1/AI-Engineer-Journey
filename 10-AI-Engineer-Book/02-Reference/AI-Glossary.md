# واژه شماره ۱

### ترجمه

سیستم مدیریت نسخه

Git

### تعریف

# AI Glossary
## فرهنگ واژگان AI Engineer Journey

---

## Repository

**معنی فارسی**

مخزن پروژه

**تعریف**

محلی که تمام فایل‌های پروژه و تاریخچه تغییرات آن در Git نگهداری می‌شود.

**مثال**

پوشه:

```
E:\AI-Engineer-Journey
```

یک Repository است.

---

## Local Repository

**معنی فارسی**

مخزن محلی

**تعریف**

نسخه‌ای از پروژه که روی کامپیوتر شخصی قرار دارد.

تمام تغییرات ابتدا در این Repository انجام می‌شوند.

---

## Remote Repository

**معنی فارسی**

مخزن راه دور (آنلاین)

**تعریف**

نسخه‌ای از پروژه که روی اینترنت (مانند GitHub) نگهداری می‌شود.

این نسخه برای پشتیبان‌گیری، اشتراک‌گذاری و همکاری استفاده می‌شود.

---

## Remote

**معنی فارسی**

اتصال به یک مخزن راه دور

**تعریف**

Remote نام ارتباط بین Repository محلی و Repository آنلاین است.

یک پروژه می‌تواند چند Remote داشته باشد.

---

## Origin

**معنی فارسی**

نام مستعار مخزن اصلی

**تعریف**

Origin نام پیش‌فرض اولین Remote در Git است.

به جای نوشتن آدرس کامل GitHub، از نام کوتاه Origin استفاده می‌کنیم.

---

## Push

**معنی فارسی**

ارسال

**تعریف**

عمل انتقال تغییرات از Repository محلی به Repository آنلاین.

---

## Fetch

**معنی فارسی**

دریافت

**تعریف**

دریافت اطلاعات و تغییرات جدید از Repository آنلاین، بدون اعمال آن‌ها روی فایل‌های پروژه.

---

## Branch

**معنی فارسی**

شاخه

**تعریف**

نسخه‌ای مستقل از پروژه که می‌توان روی آن تغییرات جدید ایجاد کرد.

هر پروژه حداقل یک Branch اصلی دارد.

---

## Master

**معنی فارسی**

شاخه اصلی

**تعریف**

نام شاخه اصلی بسیاری از پروژه‌های قدیمی Git.

در پروژه ما شاخه اصلی:

```
master
```

است.

---

## Main

**معنی فارسی**

شاخه اصلی

**تعریف**

نام جدید شاخه اصلی در بسیاری از پروژه‌های Git.

در پروژه‌های جدید معمولاً به جای Master استفاده می‌شود.

---

## Upstream

**معنی فارسی**

ارتباط پیش‌فرض

**تعریف**

ارتباطی که بین یک Branch محلی و Branch متناظر آن در Repository آنلاین برقرار می‌شود.

پس از اجرای دستور:

```bash
git push -u origin master
```

Git این ارتباط را به خاطر می‌سپارد.

---

## Git Credential Manager

**معنی فارسی**

مدیر اعتبارنامه Git

**تعریف**

ابزاری که اطلاعات ورود و مجوزهای GitHub را به صورت امن مدیریت می‌کند تا لازم نباشد هر بار نام کاربری و رمز عبور وارد شود.

---

## Authentication

**معنی فارسی**

احراز هویت

**تعریف**

فرآیندی که طی آن GitHub بررسی می‌کند آیا کاربر اجازه دسترسی به Repository را دارد یا خیر.

---

## Version Control System (VCS)

**معنی فارسی**

سیستم کنترل نسخه

**تعریف**

سیستمی که تمام تغییرات فایل‌ها را ثبت و مدیریت می‌کند و امکان بازگشت به نسخه‌های قبلی را فراهم می‌سازد.

Git یکی از معروف‌ترین سیستم‌های کنترل نسخه است.

---

## Commit

**معنی فارسی**

ثبت تغییرات

**تعریف**

ثبت یک نسخه از پروژه در تاریخچه Git.

هر Commit مانند یک عکس از وضعیت پروژه در یک لحظه مشخص است.

---

## Commit Hash

**معنی فارسی**

شناسه یکتای Commit

**تعریف**

کدی منحصربه‌فرد که Git برای شناسایی هر Commit تولید می‌کند.

این شناسه بر مبنای اعداد و حروف شانزده‌شانزدهی (Hexadecimal) نمایش داده می‌شود.

---

## GitHub

**معنی فارسی**

سرویس میزبانی پروژه‌های Git

**تعریف**

پلتفرمی مبتنی بر Git که امکان نگهداری، اشتراک‌گذاری و همکاری روی پروژه‌های نرم‌افزاری را فراهم می‌کند.
